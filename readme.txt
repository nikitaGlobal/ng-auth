Обязательное подтверждение личности пользователей (SMS/ЕСИА) при регистрации и входе. Соответствие 406-ФЗ.

== Description ==

**NG Auth** — WordPress-плагин обязательного дополнительного подтверждения личности пользователей при регистрации и первом последующем входе для ранее зарегистрированных аккаунтов без подтверждения.

Реализует механизм верификации в соответствии с требованиями Федерального закона от 31.07.2023 № 406-ФЗ через SMS (МТС Exolve, SMS Aero) или ЕСИА (Госуслуги).

= Возможности =

* Обязательная верификация при регистрации (стандартная форма, WooCommerce, REST API)
* Принудительный перевод на подтверждение существующих неподтверждённых пользователей при входе
* Настраиваемая обязательность: глобально, по ролям, по доменным зонам почты
* Провайдерная модель — три провайдера «из коробки», возможность добавления своих
* SMS-верификация: отправка OTP, валидация кода, rate limiting, блокировка по попыткам
* ЕСИА-верификация: полный OAuth 2.0 flow с порталом Госуслуг
* Пользовательский интерфейс: выбор провайдера, ввод кода, таймер повторной отправки
* Административная панель: настройки, статистика, таблица пользователей со статусами
* Отдельная таблица журнала попыток верификации
* Рассылка напоминаний неподтверждённым пользователям через cron (Email + SMS)
* WP-CLI: `wp ng-auth stats`, `wp ng-auth status`, `wp ng-auth verify`, `wp ng-auth mark-unverified`, `wp ng-auth send-reminders`
* Полная совместимость с WooCommerce (регистрация, my-account, ограничение endpoints)
* Интеграция с WordPress Personal Data Erasure / Export
* Архитектура: расширяемая система хуков и фильтров для кастомных провайдеров
* Не зависит от WooCommerce — работает на чистом WordPress
* i18n-ready: все строки интернационализированы (textdomain `ng-auth`)

= Провайдеры «из коробки» =

1. **МТС Exolve** — SMS-верификация через API МТС
2. **SMS Aero** — SMS-верификация через сервис SMS Aero
3. **ЕСИА (Госуслуги)** — подтверждение личности через OAuth 2.0 портала Госуслуг

= Архитектурные точки расширения =

* `NG_Auth_Provider_Interface` — интерфейс для кастомных провайдеров
* `ng_auth_register_providers` — фильтр регистрации провайдеров
* `ng_auth_is_mandatory_for_user` — фильтр программного управления обязательностью
* Зарезервированы точки для: VK ID, mos.ru, Yandex ID

== Installation ==

1. Загрузите папку `ng-auth` в `/wp-content/plugins/` или установите через «Плагины → Добавить новый → Загрузить плагин»
2. Активируйте плагин через меню «Плагины»
3. Перейдите в «NG Auth → Настройки» и настройте обязательность и провайдеров
4. При необходимости выполните `wp ng-auth mark-unverified` для массовой пометки существующих пользователей

= Требования =

* WordPress 6.0 или выше
* PHP 8.0 или выше
* Для SMS-провайдеров: доступ к API провайдера
* Для ЕСИА: HTTPS на сайте, зарегистрированная мнемоника в ЕСИА
* WooCommerce 8.0+ (опционально)

== Frequently Asked Questions ==

= Обязательно ли устанавливать WooCommerce? =

Нет. Плагин работает на чистом WordPress. WooCommerce-интеграция активируется автоматически при обнаружении WooCommerce.

= Как добавить свой провайдер? =

Создайте класс, реализующий `NG_Auth_Provider_Interface`, и зарегистрируйте его через хук:

`add_filter( 'ng_auth_register_providers', function( array $providers ) {
    $providers[] = new My_Custom_Provider();
    return $providers;
} );`

Подробнее в файле `docs/providers.md`.

= Как пометить существующих пользователей как неподтверждённых? =

Выполните WP-CLI команду:

`wp ng-auth mark-unverified`

= Что будет при исчерпании попыток ввода OTP? =

Аккаунт блокируется (статус `blocked`). Администратор может разблокировать через `wp ng-auth verify <user_id>`.

= Как плагин взаимодействует с законом о персональных данных? =

* Телефон хранится в хэшированном виде
* OTP-код удаляется после использования или истечения
* Поддерживается WordPress Personal Data Erasure / Export
* Журнал попыток может быть очищен через интерфейс администрирования

= Как отключить обязательность для конкретного пользователя? =

Используйте фильтр:

`add_filter( 'ng_auth_is_mandatory_for_user', function( bool $mandatory, WP_User $user ) {
    return $user->ID !== 42;
}, 10, 2 );`
